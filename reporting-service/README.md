# Reporting Service API

Подсистема отчётности предназначена для сбора событий из других сервисов аэропорта и формирования агрегированных отчётов по пассажирам и рейсам.

Сервис принимает события, сохраняет их в базе данных и поддерживает агрегированную статистику по рейсам.

---

# 1. Проверка состояния сервиса

## GET `/health`

Проверка доступности сервиса.

### Ответ

```json
{
  "status": "ok"
}
```

# 2. Приём событий

## POST /v1/events

Принимает событие от других сервисов системы и сохраняет его в базе данных.
Поддерживается защита от дубликатов по eventId.
После сохранения событие обновляет агрегированную статистику рейса.

Тело запроса
```json
{
  "eventId": "event-123",
  "eventType": "ticket_sold",
  "flightId": "SU123",
  "passengerId": "P-001",
  "count": 1,
  "timestamp": "2025-01-01T10:00:00"
}
```

Поддерживаемые типы событий:
eventType - описание
ticket_sold - покупка билета
ticket_refunded - возврат билета
passenger_checked_in - регистрация пассажира
passengers_boarded - посадка пассажиров

Ответ
```json
{
  "status": "accepted"
}
```

или

```json
{
  "status": "duplicate"
}
```
если событие уже было обработано.

# 3. Отчёт по конкретному рейсу

## GET /v1/reports/flights/{flight_id}

Возвращает агрегированную статистику по конкретному рейсу.

Ответ
```json
{
  "flight_id": "SU123",
  "tickets_sold": 120,
  "tickets_refunded": 5,
  "passengers_checked_in": 100,
  "passengers_boarded": 95
}
```

# 4. Общий отчёт по пассажирам

## GET /v1/reports/summary

Возвращает суммарную статистику по всем рейсам.

Ответ
```json
{
  "tickets_sold": 520,
  "tickets_refunded": 21,
  "passengers_checked_in": 430,
  "passengers_boarded": 415
}
```

Позволяет быстро получить общие показатели системы.

# 5. Отчёт по всем рейсам

## GET /v1/reports/flights

Возвращает статистику по всем рейсам в системе.

Ответ
```json
[
  {
    "flight_id": "SU123",
    "tickets_sold": 120,
    "tickets_refunded": 5,
    "passengers_checked_in": 100,
    "passengers_boarded": 95
  }
]
```

Используется для анализа загрузки рейсов.

# 6. Метрики рейса

## GET /v1/reports/flights/{flight_id}/metrics

Возвращает дополнительные метрики рейса.
Включает коэффициент загрузки рейса.

Ответ
```json
{
  "flightId": "SU123",
  "ticketsSold": 120,
  "passengersBoarded": 95,
  "loadFactor": 0.79
}
```

loadFactor — коэффициент загрузки рейса.

# 7. Статистика событий

## GET /v1/reports/events

Возвращает распределение событий по типам.

Ответ
```json
[
  {
    "event_type": "ticket_sold",
    "count": 520
  },
  {
    "event_type": "passenger_checked_in",
    "count": 430
  }
]
```

Используется для анализа активности системы.

# 8. Временная статистика

## GET /v1/reports/timeline

Возвращает количество событий по времени.

Группировка выполняется по часам.

Ответ
```json
[
  {
    "hour": "2025-01-01T10:00:00",
    "events": 45
  }
]
```

Позволяет строить временные графики активности аэропорта.

# 9. Последние события

## GET /v1/events/recent

Возвращает список последних событий.

Параметр limit — по умолчанию 20

Ответ
```json
[
  {
    "event_id": "event-123",
    "event_type": "ticket_sold",
    "flight_id": "SU123"
  }
]
```

Используется для отладки и мониторинга системы.

# 10. Сброс отчётов (для демо)

## POST /v1/reports/reset

Удаляет все события и статистику из базы данных.
Используется для подготовки системы к новой демонстрации.

Ответ
```json
{
  "status": "reset"
}
```


# Основная идея подсистемы

Подсистема отчётности реализует event-driven сбор статистики:
- сервисы публикуют события
- reporting service сохраняет события
- агрегирует статистику
- предоставляет отчёты через API

Это позволяет получать аналитические данные без нагрузки на основные сервисы системы.
