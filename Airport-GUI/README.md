# Airport Visualizer

Визуализатор аэродрома в реальном времени. Отображает интерактивную карту аэропорта с позициями самолётов, наземного транспорта, статусами рейсов и процессом обслуживания.

## Что отображает

- Граф аэродрома (узлы + рёбра) из `airport_map.json`
- Позиции самолётов (в воздухе и на земле)
- Позиции наземного транспорта (FollowMe, автобусы, кейтеринг)
- Статусы рейсов и сессий обслуживания
- Логи контейнеров (при наличии Docker-сокета)

## Структура проекта

```
├── data/
│   └── airport_map.json          # Карта аэродрома (узлы, рёбра)
├── services/
│   └── airport_visualizer/
│       ├── main.py               # Приложение (FastAPI + встроенный HTML/JS)
│       ├── requirements.txt      # Python-зависимости
│       ├── Dockerfile
│       └── __init__.py
├── docker-compose.yml
└── README.md
```

## Запуск

### Docker (рекомендуется)

```shell
docker compose up --build
```

Визуализатор будет доступен на **http://localhost:8087**.

### Локально (без Docker)

```shell
cd services/airport_visualizer
pip install -r requirements.txt
python main.py
```

По умолчанию слушает порт `8087`. Переопределяется через `VISUALIZER_PORT`.

## Переменные окружения

Все адреса внешних сервисов настраиваются через переменные окружения. По умолчанию указывают на `localhost`:

| Переменная                        | По умолчанию            | Описание                     |
| --------------------------------- | ----------------------- | ---------------------------- |
| `VISUALIZER_PORT`                 | `8087`                  | Порт визуализатора           |
| `GC_HOST` / `GC_PORT`             | `localhost:8081`        | Ground Control               |
| `INFO_HOST` / `INFO_PORT`         | `localhost:8082`        | Information Panel            |
| `FOLLOWME_HOST` / `FOLLOWME_PORT` | `localhost:8083`        | FollowMe                     |
| `BOARD_HOST` / `BOARD_PORT`       | `localhost:8084`        | Board                        |
| `HANDLING_HOST` / `HANDLING_PORT` | `localhost:8085`        | Handling Supervisor          |
| `BUS_HOST` / `BUS_PORT`           | `localhost:8086`        | Passenger Bus                |
| `CATERING_HOST` / `CATERING_PORT` | `localhost:8089`        | Catering Truck               |
| `GC_MAP_PATH`                     | `data/airport_map.json` | Путь к файлу карты аэродрома |

Для подключения к внешним сервисам создайте `.env` файл:

```env
GC_HOST=192.168.1.10
GC_PORT=8081
INFO_HOST=192.168.1.10
INFO_PORT=8082
# ... и т. д.
```

## API визуализатора

### Собственные эндпоинты

```
GET  /                  Главная страница — интерактивная карта аэропорта (HTML)
GET  /health            Проверка здоровья сервиса
GET  /api/map           Структура карты аэродрома (JSON из airport_map.json)
GET  /api/snapshot      Агрегированный снимок состояния всех сервисов
GET  /api/logs/tail     Логи Docker-контейнеров (?n=150&since=0)
```

### Демо-режим

```
GET  /api/demo/landing              Запуск демо-сценария посадки рейса (SSE-стрим)
GET  /api/demo/status/{sessionId}   Статус фоновой задачи демо-сценария
POST /api/demo/reset                Сброс состояния для чистой демки
```

Демо-режим создаёт рейс, регистрирует его, запускает агент самолёта, ожидает посадку и запускает наземное обслуживание — всё автоматически для наглядности.

## Внешние сервисы — какие API использует визуализатор

Визуализатор — **только потребитель** REST API. Он не имеет собственной БД и не использует RabbitMQ.

### Ground Control (порт 8081)

| Метод  | Эндпоинт                         | Для чего                                                        |
| ------ | -------------------------------- | --------------------------------------------------------------- |
| `GET`  | `/v1/visualizer/snapshot`        | Снимок состояния: карта, занятые рёбра/узлы, позиции транспорта |
| `GET`  | `/v1/visualizer/events?since=N`  | Лог событий (посадка, перемещение FollowMe и т.д.)              |
| `POST` | `/v1/vehicles/init`              | Регистрация транспорта на карте (используется в демо)           |
| `POST` | `/v1/flights/{flightId}/tookoff` | Освобождение парковки (используется в демо)                     |

### Information Panel (порт 8082)

| Метод  | Эндпоинт           | Для чего                                |
| ------ | ------------------ | --------------------------------------- |
| `GET`  | `/v1/flights`      | Список всех рейсов и их статусов        |
| `POST` | `/v1/flights/init` | Регистрация рейса (используется в демо) |

### FollowMe (порт 8083)

| Метод  | Эндпоинт            | Для чего                                                |
| ------ | ------------------- | ------------------------------------------------------- |
| `POST` | `/v1/vehicles/init` | Инициализация машин сопровождения (используется в демо) |

### Board (порт 8084)

| Метод  | Эндпоинт              | Для чего                                         |
| ------ | --------------------- | ------------------------------------------------ |
| `GET`  | `/v1/planes`          | Список активных агентов самолётов                |
| `POST` | `/v1/planes/airborne` | Создание воздушного агента (используется в демо) |
| `POST` | `/v1/planes/grounded` | Создание наземного агента (используется в демо)  |
| `POST` | `/v1/planes/stop`     | Остановка агента самолёта (используется в демо)  |

### Handling Supervisor (порт 8085)

| Метод | Эндпоинт                         | Для чего                  |
| ----- | -------------------------------- | ------------------------- |
| `GET` | `/v1/handling/tasks`             | Список задач обслуживания |
| `GET` | `/v1/handling/status/{flightId}` | Статус обслуживания рейса |

### Passenger Bus (порт 8086)

| Метод  | Эндпоинт           | Для чего                              |
| ------ | ------------------ | ------------------------------------- |
| `GET`  | `/v1/bus/vehicles` | Список автобусов и их позиции         |
| `POST` | `/v1/bus/reset`    | Сброс автобусов в начальное состояние |

### Catering Truck (порт 8089)

| Метод  | Эндпоинт              | Для чего                               |
| ------ | --------------------- | -------------------------------------- |
| `GET`  | `/v1/catering-trucks` | Список машин кейтеринга                |
| `POST` | `/v1/catering/reset`  | Сброс кейтеринга в начальное состояние |

## Подключение к полной системе аэропорта

Если все сервисы запущены в общей Docker-сети, в `docker-compose.yml` замените переменные на имена контейнеров:

```yaml
environment:
  GC_HOST: ground-control
  INFO_HOST: information-panel
  HANDLING_HOST: handling-supervisor
  BUS_HOST: passenger-bus
  FOLLOWME_HOST: followme
  BOARD_HOST: board
  CATERING_HOST: catering
```

Для чтения логов контейнеров добавьте volume:

```yaml
volumes:
  - /var/run/docker.sock:/var/run/docker.sock:ro
```
